## Featured Public Work

### [LiteQueue.NET](https://github.com/sandeepyeg/LiteQueue.NET)

A lightweight Redis-powered message queue system built with .NET 8 Web API. It explores queue creation, delayed messages, retries, dead-letter queues, pub/sub and scheduled jobs using clean architecture and background services.

**Why it matters:** it shows backend infrastructure thinking around queues, workers, retry limits and message lifecycle design.

### [Telegram Budget Assistant](https://github.com/sandeepyeg/telegram-budget-assistant)

An AI-powered personal finance assistant concept for categorizing transactions, tracking budget patterns and answering spending questions through a conversational interface.

**Why it matters:** it combines deterministic finance logic with AI-assisted interpretation, keeping computed totals grounded while using AI for language understanding.

### [DeliverCheck Platform](https://github.com/sandeepyeg/delivercheck-platform)

A delivery workflow platform template with driver-facing flows, operational structure and production-minded app organization.

**Why it matters:** it demonstrates practical full-stack workflow thinking around real users, statuses and day-to-day operations.

### [Jules Supervisor](https://github.com/sandeepyeg/jules-supervisor)

A developer tooling project focused on supervising automated coding work and keeping AI-assisted development easier to inspect and control.

**Why it matters:** it reflects how I think about AI tools: useful, powerful and still needing clear review boundaries.

### [Aspire Angular Clean Template](https://github.com/sandeepyeg/aspire-angular-clean-template)

A reusable .NET Aspire + Angular + Clean Architecture starter template with PostgreSQL, Identity scaffolding, OpenTelemetry, Serilog and layered testing projects.

**Why it matters:** it gives new projects a strong foundation with orchestration, observability and clean project boundaries from the first commit.

## AI Engineering Notes

I am especially interested in AI that makes software more useful without making systems less trustworthy.

- **RAG and document intelligence:** ingestion, embeddings, vector search, grounded answers and source citation.
- **AI agents:** task decomposition, execution history, worker orchestration, retries and observable agent workflows.
- **AI-assisted development:** codebase exploration, test planning, documentation, refactoring support and careful review.
- **Responsible boundaries:** deterministic systems should own facts, totals, permissions and irreversible actions; AI should assist where language, search and pattern recognition help.
